# zdrav.lenreg — Telegram-бот мониторинга слотов записи к врачу

Telegram-бот для отслеживания появления свободных талонов (слотов) на портале [zdrav.lenreg.ru](https://zdrav.lenreg.ru). Позволяет выбрать врача, настроить фоновый мониторинг и получать уведомления при появлении или исчезновении слотов.

## Функциональность

- **Поиск врачей по специальности** — просмотр списка специальностей и врачей в выбранной поликлинике
- **Мониторинг слотов** — фоновый, настраиваемый периодический опрос API zdrav.lenreg.ru
- **Уведомления в Telegram** — мгновенные оповещения при появлении/исчезновении слотов у отслеживаемых врачей
- **Веб-интерфейс метрик** — Prometheus-совместимый эндпоинт на `:9090/metrics`
- **Команда `/status`** — проверка состояния бота, метрик здоровья (только для администраторов)
- **Автоудаление старых сообщений** — TTL-очистка уведомлений через config
- **Поддержка трёх типов поликлиник** — взрослая, стоматологическая, детская
- **Многопользовательский режим** — регистрация нескольких пациентов на одного пользователя Telegram

## Команды бота

| Команда   | Описание                                                             | Доступ |
| --------- | -------------------------------------------------------------------- | ------ |
| `/start`  | Начало работы, вывод главного меню со списком пациентов              | Все    |
| `/status` | Состояние мониторинга, метрики здоровья (активные задачи, Redis, БД) | Админы |

### Интерактивные действия (callback-кнопки)

Помимо текстовых команд, весь основной функционал доступен через инлайн-кнопки:

- **Выбор пациента** — `sel_p_{patient_id}` — переключение между зарегистрированными пациентами
- **Выбор города** — `sel_cty_{city_idx}` — фильтрация клиник по городу
- **Выбор клиники** — `sel_c_{clinic_id}` — просмотр врачей в конкретной поликлинике
- **Включение/отключение мониторинга** — `tgl_{doctor_id}` — переключение отслеживания врача
- **Остановка мониторинга по пациенту** — `stop_patient_{patient_id}`
- **Остановка мониторинга по клинике** — `stop_clinic_{clinic_id}`
- **Остановка всего мониторинга** — `stop_all`
- **Удаление пациента** — `del_p_{patient_id}` — удаление данных пациента из системы
- **Добавление пациента** — `start_add_p` — запуск регистрации нового пациента
- **Навигация** — `back_to_main`, `back_to_cities_{...}`, `back_to_clinics_{...}`
- **Отмена регистрации** — `cancel_registration`, `skip_alias`

## Архитектура

Проект построен по модульной слоёной архитектуре на базе `aiogram 3.x`.

```text
src/
├── api/            # HTTP-клиент zdrav.lenreg.ru (httpx, Pydantic-валидация)
├── assets/         # Изображения для навигации и уведомлений
├── database/       # SQLite (WAL-режим) + миграции + кэш-менеджер
├── filters/        # Кастомные фильтры (IsAdmin)
├── handlers/       # Хендлеры команд и callback-запросов
├── keyboards/      # Inline-клавиатуры
├── middleware/     # Rate limit, ActivityLog, ErrorBoundary, UserData
├── services/       # Мониторинг, healthcheck, метрики, discovery, cleanup
├── utils/          # Redis, кэш, логирование, хелперы, proxy_discovery
└── main.py         # Точка входа, DI, инициализация
```

### Ключевые технологии

| Компонент              | Технология                                     |
| ---------------------- | ---------------------------------------------- |
| Telegram Bot Framework | `aiogram 3.x`                                  |
| HTTP-клиент            | `httpx` + `aiohttp-socks` (SOCKS5 proxy)       |
| База данных            | SQLite (WAL-режим, `aiosqlite`)                |
| Кэш / FSM-хранилище    | Redis (через `redis-py`)                       |
| Rate Limiting          | `aiolimiter`                                   |
| Метрики                | Prometheus-совместимый HTTP-сервер (`aiohttp`) |
| Валидация              | Pydantic (модели API-ответов)                  |
| Конфигурация           | `pydantic-settings` + `.env` + БД              |
| Логирование            | `loguru`                                       |
| Мониторинг ошибок      | Sentry + NTFY push-уведомления                 |

Полная архитектура и граф зависимостей: [`specs/ARCHITECTURE.md`](specs/ARCHITECTURE.md).

## Быстрый старт

### 1. Клонирование

```bash
git clone https://github.com/acidmsg/lenreg_ticket_bot.git
cd zdrav.lenreg
```

### 2. Настройка окружения

Скопируйте [`.env.example`](.env.example) в `.env` и заполните обязательные переменные:

```bash
cp .env.example .env
```

Минимально необходимые настройки:

| Переменная                   | Описание                                                 |
| ---------------------------- | -------------------------------------------------------- |
| `BOT_TOKEN`                  | Токен Telegram-бота (получить у BotFather)               |
| `PROXY_URL`                  | SOCKS5-прокси (например, `socks5://user:pass@host:port`) |
| `ADMIN_IDS`                  | Telegram ID администраторов (через запятую)              |
| `DISCOVERY_PATIENT_ID_ADULT` | ID взрослого пациента для discovery врачей               |
| `DISCOVERY_PATIENT_ID_CHILD` | ID детского пациента для discovery врачей                |
| `CSRF_TOKEN`                 | CSRF-токен для запросов к zdrav.lenreg.ru                |

Полный список переменных — в [`.env.example`](.env.example).

### 3. Установка зависимостей

```bash
pip install -r requirements.txt
```

Либо через Poetry (если установлен):

```bash
poetry install
```

### 4. Запуск Redis

**Локально (если установлен):**

```bash
redis-server
```

**Через Docker:**

```bash
docker run -d --name zdrav_redis -p 6379:6379 redis:7-alpine
```

### 5. Запуск бота

```bash
python -m src.main
```

## Docker-развёртывание

Запуск одной командой (требуется настроенный `.env`):

```bash
docker compose up -d
```

Сервисы, определённые в [`docker-compose.yml`](docker-compose.yml):

| Сервис  | Назначение             | Порты            |
| ------- | ---------------------- | ---------------- |
| `redis` | FSM-хранилище и кэш    | `127.0.0.1:6379` |
| `bot`   | Telegram-бот (polling) | `127.0.0.1:9090` |

- Порт `9090` — Prometheus-метрики (закреплён только за localhost)
- Данные БД (`data/`) и Redis (`/data`) сохраняются в Docker-томах

Остановка:

```bash
docker compose down
```

## Troubleshooting Guide

### Бот не запускается

1. Проверьте наличие и корректность `.env` файла — все обязательные переменные должны быть заполнены
2. Убедитесь, что `BOT_TOKEN` получен у [BotFather](https://t.me/BotFather) и активен
3. Проверьте доступность прокси: `curl --socks5-hostname host:port https://api.telegram.org`
4. Проверьте логи на наличие ошибок подключения к Redis (`REDIS_URL`)

### Нет слотов / уведомлений

1. Проверьте доступность zdrav.lenreg.ru через ваш прокси/VPN
2. Убедитесь, что вы корректно зарегистрировали пациента (команда `/start`, затем кнопка «Добавить пациента»)
3. Проверьте, что мониторинг включён для нужного врача (зелёный индикатор 🔵 рядом с именем)
4. Если слотов действительно нет — бот ожидает их появления, это нормальное поведение

### Ошибки валидации API

Если в логах появляются `ValidationError` — структура ответов API zdrav.lenreg.ru могла измениться. Попробуйте отключить валидацию в `.env`:

```env
API_VALIDATE_RESPONSES=False
```

После исправления совместимости включите валидацию обратно.

### Redis недоступен

1. Проверьте, запущен ли Redis: `redis-cli ping` (должен ответить `PONG`)
2. Проверьте `REDIS_URL` в `.env` — по умолчанию `redis://localhost:6379/0`
3. Убедитесь, что порт 6379 не занят другим процессом

### SQLite блокировки

Бот использует WAL-режим SQLite (`PRAGMA journal_mode=WAL`), который минимизирует блокировки при конкурентном чтении/записи. Если вы видите `database is locked`:

1. Убедитесь, что только один процесс бота использует БД
2. Проверьте файловую систему — достаточно ли места на диске

## FAQ

### Как добавить врача для отслеживания?

1. Отправьте `/start` боту
2. Выберите пациента из списка
3. Выберите город и клинику
4. Нажмите на врача, чтобы включить мониторинг (🔵 — включено, ⚪ — выключено)

### Какой интервал мониторинга?

По умолчанию — 60 секунд. Интервал настраивается в БД (таблица `config`, ключ `monitor_interval`).

### Как остановить мониторинг конкретного врача?

В списке врачей нажмите на врача с 🔵 — мониторинг отключится. Для массовой остановки используйте кнопки «Остановить всё» или «Остановить по клинике».

### Где хранятся данные?

- **SQLite БД:** `data/bot.db` — пациенты, врачи, настройки мониторинга
- **Redis:** FSM-состояния, кэш API-запросов
- **Кэш мониторинга:** `data/monitoring_cache.json`

### Как посмотреть метрики?

Откройте в браузере: `http://localhost:9090/metrics`. Эндпоинт доступен только с localhost.

### Как обновить данные о врачах?

Бот автоматически обновляет список врачей при первом обращении к клинике (on-demand discovery). Принудительное обновление — перезапустите бота.

### Как удалить пациента?

В главном меню выберите пациента, затем нажмите «Удалить». Данные о пациенте и его мониторинг будут удалены.
