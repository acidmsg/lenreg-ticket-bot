## Часть 4 — Автоудаление сообщений (TTL) и единый механизм удаления

### Задача

Настроить автоудаление сообщений для всех пользователей спустя время (7 дней), чтобы чат не засорялся старыми уведомлениями.

### Реализация

#### 1. Конфигурация (`config.py`)

```python
# Автоудаление сообщений: TTL в секундах (7 дней по умолчанию)
MESSAGE_TTL_SECONDS: int = 604800
CLEANUP_INTERVAL: int = 3600  # Проверять каждые 1 час
```

#### 2. Фоновая задача (`services/cleanup.py`)

- Цикл проверяет `last_messages` всех пользователей раз в час
- Удаляет сообщения, возраст которых превышает `MESSAGE_TTL_SECONDS`
- Пропускает записи в старом формате (int без timestamp) — они конвертируются при первом обновлении
- Graceful shutdown через `asyncio.CancelledError`

#### 3. Обновление `database/manager.py`

`set_last_message_id()` теперь сохраняет:

```python
{"msg_id": message_id, "ts": time.time()}
```

`get_last_message_id()` читает оба формата:

- Новый `dict` → `value.get("msg_id")`
- Старый `int` → `value`

#### 4. Единый механизм удаления (`utils/helpers.py` + `handlers/common.py`)

Введены единые функции:

| Функция | Назначение |
|---------|-----------|
| `extract_msg_id()` | Разбор `last_messages` (int/dict) → int | None |
| `_delete_cleanup_msg_entry()` | Удалить одно сообщение по ключу |
| `_delete_cleanup_msg_entries()` | Удалить все сообщения с префиксом ключа |

Все сценарии используют их:

| Сценарий | Вызов | Файл |
|----------|-------|------|
| Автоудаление по TTL | `extract_msg_id()` + `value.get("ts")` | `cleanup.py` |
| Отключение врача | `_delete_cleanup_msg_entry(bot, uid, p_id_d_id, ...)` | `handlers/common.py: toggle_doctor` |
| Стоп пациента | `_delete_cleanup_msg_entries(bot, uid, p_id_, ...)` | `handlers/common.py: stop_patient_monitoring` |
| Стоп клиники | `_delete_cleanup_msg_entries(bot, uid, p_id_d_id, ...)` | `handlers/common.py: stop_clinic_monitoring` |
| Стоп всего | `_delete_cleanup_msg_entries(bot, uid, "", ...)` | `handlers/common.py: stop_all_monitoring` |

#### 5. Запуск в `main.py`

```python
background_tasks.append(asyncio.create_task(cleanup_loop(bot, db)))
```

### Проблема: SOCKS-прокси в httpx

При запуске бота `doctor_discovery` падает с ошибкой:

```
Unknown scheme for proxy URL URL('socks4://127.0.0.1:10808')
```

**Причина:** `httpx` не поддерживает схему `socks4://`. В `.env` указан `PROXY_URL=socks4://127.0.0.1:10808`, который используется aiogram через `AiohttpSession`, но `httpx.AsyncClient` в `zdrav_client.py` подхватывает переменные окружения `HTTP_PROXY`/`HTTPS_PROXY` и пытается использовать `socks4://`, что вызывает ошибку.

**Решение:** исключить `socks4://` из переменных окружения для httpx-клиента или заменить на `socks5://` если прокси поддерживает SOCKS5.
