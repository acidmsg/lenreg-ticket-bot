### 3. Healthcheck-модуль (`services/healthcheck.py`)

**Ключевые компоненты:**

- `HealthMetrics` — dataclass-сборщик метрик:
  - Uptime бота
  - Статистика API (успехи/ошибки)
  - Состояние фоновых задач
  - Последняя ошибка
- `healthcheck_loop()` — фоновый цикл проверки API каждые `CHECK_INTERVAL` (5 мин)
- `format_status_report()` — форматирование отчёта для `/status`
- `metrics` — глобальный синглтон для доступа из других модулей

### 4. Команда `/status` (`handlers/common.py`)

Выводит пользователю:

- Аптайм бота
- Статистику пользователей, пациентов, мониторинга
- Состояние API с цветовой индикацией
- Живы ли фоновые задачи
- Текущие настройки
- Последняя ошибка

### 5. Интеграция в `main.py`

- `healthcheck_loop` запускается как фоновая задача
- Discovery-задачи регистрируются в `metrics.discovery_tasks_alive`
