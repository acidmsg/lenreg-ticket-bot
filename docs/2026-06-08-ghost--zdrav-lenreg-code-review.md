# 🔬 Deep Research: zdrav.lenreg

> Telegram-бот для мониторинга талонов zdrav.lenreg.ru (Ленинградская область).
> Aiogram 3.x, Python 3.11+, Docker, Redis, Qdrant, SQLite, FastAPI.
> Исходники: `/srv/bots/zdrav.lenreg/`

## Общая оценка

Проект **рабочий**, функциональность покрывает бизнес-логику, тесты есть. Но кодовая база несёт следы быстрой итеративной разработки AI-ассистентами: дублирование, мёртвый код, архитектурные компромиссы.

---

## 🔴 Критические проблемы

### 1. Безопасность: Mini App без верификации hash

**`src/handlers/mini_app.py`** — данные от Telegram Mini App принимаются, но **не проверяется подпись**. Любой может подделать web_app_data.

### 2. Инфраструктура: Docker HEALTHCHECK не работает

Redis требует пароль, но healthcheck делает `redis-cli -h redis ping` без `-a`. Контейнер будет постоянно перезапускаться.

### 3. Безопасность: `ENVIRONMENT=development` отключает аутентификацию

Если в production случайно попадёт `ENVIRONMENT=development`, весь Mini App API становится открытым без предупреждения.

### 4. Баг: `REDIS_URL` в `config.py` содержит мусор

Вместо `redis://...` там текст на русском. Redis-клиент упадёт при старте.

### 5. Баг: `CONFIG_KEY_CSRF_TOKEN` = `"***"`

Константа-ключ содержит звёздочки вместо `"csrf_token"`.

### 6. Баг: FSM-регистрация без таймаута

Если пользователь бросил регистрацию на середине — стейт висит в Redis/Memory навсегда. Утечка.

### 7. Баг: `delete_patient` без транзакции

Три отдельных DELETE без BEGIN/COMMIT. При сбое — сиротские записи.

### 8. Баг: `StopPatientMonitoring.city_idx = ""`

Дефолт — пустая строка, но везде по коду используется `"all"`. Конкретный баг в логике остановки мониторинга.

## 🟠 Архитектурные проблемы

### God-функция `main()` — 200+ строк

Инициализация БД, прокси, логов, кэша, веб-дашборда — всё в одной функции. Разбить на `bootstrap_*()`.

### God-класс `Database` — 1050 строк

Всё в одном: коннект, миграции, CRUD пользователей, врачей, клиник, алиасов, логов, статистики, поиск. Нарушение SRP.

### `ZdravClient`: retry-логика скопирована 5 раз

~250 строк дублирующегося кода в 5 методах. Должен быть `_request_with_retry()`.

### Перекрёстные импорты приватных `_`-символов

`monitor.py` импортирует `from healthcheck import _safe_set, _metrics_lock`. Приватные имена не должны быть публичным API.

### `dependencies.py` — мёртвый модуль

3 функции, которые никто не использует. Все роутеры ходят в `request.app.state.db` напрямую.

### `mini_app.py` — заглушка

Принимает данные от Mini App, но не сохраняет их в БД. Функциональный разрыв: Mini App и бот живут в разных вселенных.

### `force_check_doctor` — монстр на 120 строк

Дублирует логику из `monitor.py`. Должен делегировать в сервисный слой.

## ✅ Что сделано хорошо

- **Graceful degradation** — бот работает без Redis
- **PII-маскирование** в логах
- **HMAC-верификация initData** — корректная, с проверкой `auth_date` и защиты от future
- **Per-event-loop RedisClient** — поддерживает бота + веб-дашборд
- **Тесты с моками** — мониторинг, кэш, database manager
- **i18n** — интернационализация заложена с самого начала
