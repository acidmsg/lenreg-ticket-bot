# Системные стандарты: Python и Markdown

Применяются ко **всем** текущим и будущим задачам в проекте `zdrav.lenreg`.

---

## Python (.py)

### Типизация

- Обязательные type hints для **всех** сигнатур функций и методов.
- Отсутствие ошибок mypy (strict-опционально, но `--check-untyped-defs` минимум).
- Аннотации для переменных уровня модуля.

### Форматирование

- Жёсткое форматирование через Ruff (линтер + форматтер) и Black-совместимый стиль.
- Конфигурация Ruff в [`pyproject.toml`](pyproject.toml) задаёт стиль кавычек (`"`), отступы (4 пробела), целевую версию Python.

### Импорты

- **Абсолютные** предпочтительнее относительных.
- Префикс `src.` для всех внутренних импортов (например, `from src.config import settings`).
- Строгая сортировка: стандартная библиотека → сторонние → внутренние (isort-совместимо, через Ruff).

### Прочее

- PEP-8: 4 пробела, 100 символов на строку, snake_case для функций/переменных, PascalCase для классов.
- Докстринги: тройные двойные кавычки `"""`, первая строка — краткое описание.

## Markdown (.md)

### Иерархия заголовков

- **Один H1** (`#`) на файл.
- Строго последовательная иерархия: H1 → H2 → H3 (без пропусков уровней).

### Блоки кода

- Обязательное указание языка: `python`, `json`, `bash`, `text`, `mermaid`.

### Списки

- Единый маркер: `-`.
- Пустая строка перед и после списка.

### Пробелы

- Отсутствие висячих пробелов в конце строк.
- Завершение файла ровно одной пустой строкой (`\n` в конце).

### Валидация

- Файлы должны проходить `markdownlint` и `prettier`.
- Форматирование через `npx prettier --write` (proseWrap: preserve, printWidth: 120).
- Проверка markdownlint: `npx markdownlint "docs/**/*.md" ".roo/**/*.md" "*.md"` (ожидается 0 errors).

---

## Стилистика текста

- Лаконичность, сухость, технический стиль.
- Исключены: эмоции, избыточные описания, вода.
- Только факты, алгоритмы, спецификации.

---

## Ссылки на файлы

- В markdown-документах ссылки на файлы оформляются с указанием строки: `[`filename.ext:line`](relative/path/file.ext:line)`.
- Для конструкций кода указывается и строка: ``[`function_name()`](path/file.py:42)``.
