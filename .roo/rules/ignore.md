# Игнорируемые файлы и директории

Чтобы не тратить контекст (токены) на бесполезные файлы, Cline должен **игнорировать** следующие директории и файлы — не читать их, не листинговать рекурсивно, не анализировать содержимое.

## Обязательно игнорировать (полностью)

| Директория | Причина |
|---|---|
| `.venv/` | Виртуальное окружение Python (~4 500 файлов, 51 MB) |
| `__pycache__/` | Кэш байткода Python |
| `.history/` | История изменений VSCode (338 файлов) |
| `.vscode/` | Настройки редактора |
| `logs/` | Файлы логов времени исполнения |
| `data/` | Рантайм данные (бд, кэш и т.п.) |
| `.pytest_cache/` | Кэш тестов |
| `.git/` | Git-объекты, коммиты и т.п. |

## Файлы, которые не нужно читать без явной необходимости

| Файл | Причина |
|---|---|
| `docs/SESSION_2026-*.md` | Исторические логи сессий (только если явно запрошено) |
| `docs/inline_review_report.md` | Отчёты ревью (только если явно запрошено) |
| `SESSION_LOG.md` | Лог сессий накапливается — читать только последние записи |
| `z` | Какой-то бинарный/служебный файл |

## Принцип

1. Если файл/директория есть в списке выше — **не читай его** без явного указания пользователя.
2. Если пользователь явно говорит «почитай файл X» — конечно, читай.
3. При старте сессии не трать токены на перечисление содержимого `.venv/`, `.history/` и т.п.

## Синтаксис для самоограничения

При анализе проекта:

- Не делай `list_files` (рекурсивно) для `.venv/`, `.history/`, `logs/`, `data/`
- Если нужно понять структуру проекта — используй `list_files` на корень без рекурсии
- Для поиска по коду используй `search_files` с указанием конкретной директории (например, `path: "src"`), а не всего проекта
